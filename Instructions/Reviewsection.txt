PRD: Testimonials Section with Vertical Auto-Scrolling Carousel
Reference
Live example: https://landscapers.framer.website/ — the testimonials section with the vertical auto-scrolling review cards on the right side.

Overview
Build a full-width testimonials section that has a background image with a dark gradient overlay, a left column with heading/CTA/review-platform links, and a right column containing a vertical infinite-scrolling carousel of testimonial cards. The cards scroll upward continuously on a loop with no user interaction required.

Tech Stack

Next.js (App Router, "use client" component)
framer-motion for the vertical infinite scroll animation
next/image for optimized images
Tailwind CSS for styling (or CSS modules — your choice)
No other dependencies

Right Column — Vertical Auto-Scrolling Carousel
This is the main interactive element. It is a vertical infinite-scrolling list of testimonial cards that moves upward automatically.
Animation Mechanism
How the infinite scroll works:

Take the array of testimonials (e.g. 3 items)
Duplicate the array 3× so you have 9+ items rendered in a vertical flex column
Animate the <ul> container's translateY from 0 to -(height of one set of items) using a CSS animation or framer-motion
When the animation completes one cycle, it resets seamlessly back to 0 (because the duplicated items make the loop seamless)
The animation is continuous, linear, and infinite — no easing, no pauses

Implementation approach (CSS animation — simplest):
css@keyframes scroll-up {
  0% { transform: translateY(0); }
  100% { transform: translateY(calc(-1 * var(--scroll-distance))); }
}
Where --scroll-distance equals the combined height of one set of testimonial items + gaps.
Alternatively use framer-motion's animate with repeat: Infinity and ease: "linear".
Container styling:

overflow: hidden on the outer <section> wrapper
The <ul> inside uses display: flex, flex-direction: column, gap: 20px
will-change: transform for GPU acceleration
Height of the visible area should be constrained (e.g. max-height: 500px or match the left column height)

Pause on hover (optional but recommended):

On mouseenter of the carousel area, pause the animation (animation-play-state: paused)
On mouseleave, resume

Responsive Behavior
Desktop (>1024px):

Two-column layout, side by side
Carousel height matches left column height naturally
Cards are full width within the right column

Tablet (768–1024px):

Two columns can shrink, reduce padding
Carousel may show 2 visible cards at a time

Mobile (<768px):

Stack vertically: heading/CTA on top, carousel below
Carousel gets a fixed height (~400px) with overflow hidden
Cards stretch full width
Review platform cards stack or shrink


Acceptance Criteria

 Background image renders full-bleed with gradient overlay darkening left side
 Left column: label pill, heading, CTA button, and review platform links all render correctly
 CTA button has the two-part design (text section + icon section) with orange styling
 Review platform links open in new tabs
 Testimonial cards scroll upward continuously and infinitely with no visible seam/jump
 Scroll animation is linear (no easing) and smooth (GPU-accelerated via transform)
 Cards use glassmorphism: backdrop-filter: blur(5px) + semi-transparent white background
 Each card displays: avatar, name, location, review text, divider, source label, and star rating
 Section is fully responsive — stacks on mobile
 Carousel pauses on hover (optional but recommended)
 All text colors match the token table above
 Component is "use client" with no SSR issues


Edge Cases

Single testimonial: Duplicate it enough times to fill the visible height + one extra set for the loop
Very long review text: Allow text to wrap naturally; card height grows. The scroll animation should use dynamically calculated height (measure with a ref after render)
No background image: Fall back to a solid dark color (#0a0a0a)
Missing avatar: Show a placeholder circle with the person's initials


Optional Enhancements (not in v1 scope)

Scroll speed prop (configurable animation duration)
Reverse scroll direction (downward)
Click on card to expand/read full review in a modal
Fetch testimonials from a CMS or API instead of static props
Fade-out mask at top and bottom edges of the carousel (using a CSS gradient mask) for a polished look
